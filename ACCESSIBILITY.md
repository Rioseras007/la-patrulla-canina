# Política de Accesibilidad (A11Y)\n\n## Compromiso\n\nLa Patrulla Canina se compromete a ser accesible para todos los usuarios, independientemente de sus capacidades o el dispositivo que utilicen.\n\n## Estándares\n\nNuestro proyecto cumple con los siguientes estándares:\n\n- **WCAG 2.1 Nivel AA:** Web Content Accessibility Guidelines\n- **Aria 1.2:** Accessible Rich Internet Applications\n- **HTML 5:** Semántica correcta\n- **Keyboard Navigation:** Accesibilidad por teclado\n\n## Requisitos Implementados\n\n### 1. Contraste de Colores\n\n✅ **Cumplimiento:** WCAG AA (4.5:1 para texto)\n\n```\nColor Primario (#854CE6) sobre Fondo (#050709)\nRatio: 6.8:1 ✓ Conforme\n\nCyan (#00F2FF) sobre Fondo (#050709)  \nRatio: 7.2:1 ✓ Conforme\n\nOro (#FFD700) sobre Fondo (#050709)\nRatio: 8.1:1 ✓ Conforme\n```\n\n### 2. Navegación por Teclado\n\n✅ **Totalmente funcional sin ratón**\n\n**Combinaciones de teclas:**\n- `Tab` - Navegación entre elementos\n- `Shift + Tab` - Navegación inversa\n- `Enter` / `Space` - Activar botones\n- `Escape` - Cerrar modales\n\n### 3. Semántica HTML\n\n✅ **Estructura semántica apropiada**\n\n```html\n<!-- Correcto -->\n<header>\n    <h1>La Patrulla Canina</h1>\n    <nav>...</nav>\n</header>\n\n<main>\n    <section>\n        <h2>Reproductor</h2>\n        <button>Reproducir</button>\n    </section>\n</main>\n\n<footer>...</footer>\n```\n\n### 4. ARIA Labels\n\n✅ **Atributos ARIA para elementos complejos**\n\n```html\n<button \n    aria-label=\"Reproducir pista actual\"\n    aria-controls=\"player\"\n    aria-pressed=\"false\">\n    <i class=\"fas fa-play\"></i>\n</button>\n\n<div \n    id=\"lyrics\"\n    aria-live=\"polite\"\n    aria-label=\"Letras sincronizadas\">\n    Contenido dinámico\n</div>\n```\n\n### 5. Focus Visible\n\n✅ **Indicadores de focus claros**\n\n```css\nbutton:focus {\n    outline: 3px solid #00F2FF;  /* Cyan claramente visible */\n    outline-offset: 2px;\n}\n```\n\n### 6. Responsive Design\n\n✅ **Funciona en todos los tamaños**\n\n- Móvil: 320px y superior\n- Tablet: 768px y superior  \n- Desktop: 1024px y superior\n- Ultra-wide: 1920px y superior\n\n### 7. Lenguaje Clara\n\n✅ **Contenido comprensible**\n\n- Frases cortas y claras\n- Terminología consistente\n- Explicaciones cuando sea necesario\n- Definición de abreviaturas\n\n## Problemas Conocidos\n\n### 1. Reproductor de Audio\n**Estado:** ✅ Accesible\n- Controles HTML5 nativos\n- Labels ARIA completos\n- Teclado completamente funcional\n\n### 2. Visualizador\n**Estado:** ⚠️ Decorativo\n- Solo visual, no afecta contenido\n- Información disponible sin visualizador\n\n### 3. Animaciones\n**Estado:** ✅ Respeta preferencias\n- Respeta `prefers-reduced-motion`\n- Puede desactivarse sin perder funcionalidad\n\n## Cómo Testear Accesibilidad\n\n### Herramientas Recomendadas\n\n1. **axe DevTools**\n   - Extensión Chrome/Firefox\n   - Escaneo automático de A11Y\n\n2. **Wave**\n   - Visualiza estructura de página\n   - Identifica problemas de contraste\n\n3. **Lighthouse**\n   - Integrado en Chrome Dev Tools\n   - Score de accesibilidad\n\n4. **Screen Readers**\n   - NVDA (Windows - gratuito)\n   - JAWS (Windows - comercial)\n   - VoiceOver (macOS/iOS - integrado)\n\n### Checklist Manual\n\n```\n□ Navegación completa por Tab\n□ Focus visible en todos los elementos\n□ Botones activables por Enter/Space\n□ Modales cerrable con Escape\n□ Colores no son la única información\n□ Alt text en todas las imágenes\n□ Formatos legibles en contraste\n□ Video con subtítulos (si aplica)\n□ Contenido en orden lógico\n□ Errores claramente identificados\n```\n\n## Reportar Problemas de Accesibilidad\n\nSi encuentras un problema de accesibilidad:\n\n1. **Describe el problema** con detalle\n2. **Incluye navegador y asistencia** (screen reader, etc.)\n3. **Pasos para reproducir**\n4. **Comportamiento esperado vs. actual**\n\n**Email:** accesibilidad@lapatrullacanina.com\n\n## Mejoras Futuras\n\nPlanificado:\n- [ ] Soporte para Dark Mode automático\n- [ ] Subtítulos en videos\n- [ ] Transcripciones de audio\n- [ ] Testing con usuarios reales\n- [ ] Certificación WCAG AAA\n\n## Recursos\n\n### Aprender Más\n- [WCAG 2.1 Oficial](https://www.w3.org/WAI/WCAG21/quickref/)\n- [ARIA Authoring Practices](https://www.w3.org/WAI/ARIA/apg/)\n- [MDN Accessibility](https://developer.mozilla.org/es/docs/Web/Accessibility)\n\n### Herramientas Útiles\n- [Color Contrast Checker](https://webaim.org/resources/contrastchecker/)\n- [Screen Reader Test](https://www.accessibility-checker.com/)\n- [Keyboard Navigation Test](https://www.w3.org/WAI/test-evaluate/)\n\n---\n\n**Compromiso de Accesibilidad:** Mejoranza continua\n\n**Última revisión:** Marzo 2026\n"
